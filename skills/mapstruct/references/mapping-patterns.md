# MapStruct Patterns

## Field rename

```java
@Mapping(source = "fullName", target = "name")
UserResponse toResponse(User user);
```

## Nested

```java
@Mapping(source = "address.city", target = "city")
@Mapping(source = "address.zipCode", target = "zip")
UserResponse toResponse(User user);
```

## Custom expression

```java
@Mapping(target = "displayName", expression = "java(user.getFirstName() + \" \" + user.getLastName())")
UserResponse toResponse(User user);
```

Keep expressions trivial. If complex, use `@AfterMapping`.

## qualifiedByName

```java
@Mapper(componentModel = "spring")
public interface UserMapper {

    @Mapping(source = "passwordHash", target = "password", qualifiedByName = "decryptPassword")
    UserDto toDto(User user);

    @Named("decryptPassword")
    default String decryptPassword(String hash) {
        // never decrypt in production — example only
        return "***";
    }
}
```

## @AfterMapping for post-processing

```java
@Mapper(componentModel = "spring")
public abstract class UserMapper {

    @Autowired protected ClockProvider clockProvider;

    public abstract UserResponse toResponse(User user);

    @AfterMapping
    protected void enrichTimestamps(@MappingTarget UserResponse response, User user) {
        response.setAge(Period.between(user.getBirthDate(),
                                        LocalDate.now(clockProvider.getClock())).getYears());
    }
}
```

## Partial update — @MappingTarget + ignore nulls

```java
@Mapping(target = "id", ignore = true)
@Mapping(target = "createdAt", ignore = true)
@BeanMapping(nullValuePropertyMappingStrategy = NullValuePropertyMappingStrategy.IGNORE)
void update(UpdateUserRequest req, @MappingTarget User user);
```

Null fields in `req` won't overwrite values in `user`.

## Multiple mappers — @MapperConfig

```java
@MapperConfig(componentModel = "spring",
              unmappedTargetPolicy = ReportingPolicy.ERROR,
              nullValueCheckStrategy = NullValueCheckStrategy.ALWAYS)
public interface CentralConfig {}

@Mapper(config = CentralConfig.class)
public interface UserMapper { ... }

@Mapper(config = CentralConfig.class)
public interface OrderMapper { ... }
```

## Collections

Built-in for `List<X> → List<Y>`:

```java
List<UserResponse> toResponses(List<User> users);
```

For `Set`, `Map`, custom collections — same automatic mapping if `toResponse(X)` defined.

## Generated impl location

- Maven: `target/generated-sources/annotations/`
- Gradle: `build/generated/sources/annotationProcessor/java/main/`

Add to `.gitignore`.

## Reviewing generated impl

Phase 5 reviewer-mapping reads the generated impl and flags issues:

```java
// Generated by MapStruct
@Generated(...)
@Component
public class UserMapperImpl implements UserMapper {

    @Override
    public User toEntity(CreateUserRequest req) {
        if (req == null) return null;
        User user = new User();
        user.setName(req.getName());
        user.setEmail(req.getEmail());
        return user;
    }
}
```

Check for missing fields (typo in @Mapping target = "...").
